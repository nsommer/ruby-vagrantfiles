Consistent environments for Ruby apps with Vagrant

Ruby is an interpreted, object oriented scripting language that runs on most major operating systems. However, applications might still depend on specific ruby versions or other system libraries which makes it desirable to create a consistend and isolated environment to run your ruby apps.

To achieve this, the popular tools Vagrant and rbenv can be used. Vagrant can be used to setup and run virtual machines easily. rbenv is used to manage multiple specific versions of ruby.

What the repository provides

This repository provides skripts that set up virtual machines using Vagrant and configures rbenv for your application.

Standard ruby apps

If you develop a simple ruby application, place the content of ruby/ in your applications root directory. It sets up an Ubuntu virtual machine using Vagrant and installs the current version of rbenv in it. After that, it downloads and sets up the version of the ruby interpreter that is specified in the .ruby-version file. If you require a different version of ruby, adjust the version number in the file to your needs.

Ruby on Rails apps

If you develop a ruby application that uses the popular Ruby on Rails framework, place the content of rails/ into your applications root directory. The provisioning files there basically do the same as the ones for simple ruby apps. However, they do some additional stuff useful for rails development. The http port 3000 which is the standard port of the built in rails webserver is forwarded to the host machine. Rails, SQLite and a javascript engine (nodejs) are being preinstalled. Every time the virtual machine starts, all missing gems from the Gemfile in /vagrant are installed and the built in rails webserver is started.

To start development type:

vagrant up
tail -f /log/development.log
